## Workflow Process

### 1. Webhook Trigger
- **Path**: `/check-airtable-record`
- **Purpose**: This is the entry point of the workflow, triggered by an incoming webhook request. It expects query parameters for `base_id`, `table_id`, and `formula` to perform an Airtable search.

### 2. Airtable Search Node
- **Type**: Airtable Node
- **Purpose**: Searches an Airtable base for records matching a given formula.
- **Parameters**:
  - **Base**: Dynamically set using the `base_id` from the webhook query: `={{ $('Webhook').first().json.query.base_id }}`
  - **Table**: Dynamically set using the `table_id` from the webhook query: `={{ $('Webhook').first().json.query.table_id }}`
  - **Operation**: `search`
  - **Filter By Formula**: Dynamically set using the `formula` from the webhook query: `={{ $('Webhook').first().json.query.formula }}`
- **Authentication**: Airtable OAuth2 API (Credential ID: `jBHiG9dORxXca5We`, Name: `Acee's Connection`)

### 3. If Node
- **Type**: If Node
- **Purpose**: Checks if the Airtable search returned any records.
- **Conditions**:
  - **Left Value**: `={{ $json.id }}`
  - **Operator**: `exists` (checks if the ID field exists, implying a record was found)
  - **Right Value**: (empty)
  - **Combinator**: `and`

### 4. Respond to Webhook Node
- **Type**: Respond to Webhook Node
- **Purpose**: If the "If" node condition is met (meaning records were found), this node sends back all the incoming items (the Airtable search results) as the webhook response.

### 5. Error Node
- **Type**: Respond to Webhook Node
- **Purpose**: If the "If" node condition is not met (meaning no records were found), this node sends back a "no data" response with an HTTP status code of 400, indicating the record was not found.
